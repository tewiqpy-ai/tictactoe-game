# Auto-Updates: GitHub Releases + In-App Update Checker

## Context

Capacitor Android-приложение "Игры для тиктокерских" (com.tictactoe.tiktoker, v1.0.0). Нужно автоматизировать релизы APK через GitHub Actions и добавить проверку обновлений прямо из приложения.

> **Блокер:** `gh auth` невалиден — нужно переавторизоваться (`gh auth login -h github.com`).

---

## Part 1: GitHub Actions — Автосборка APK + публикация релиза

### Шаг 1: Создать `.github/workflows/release.yml`

Триггер: пуши тега `v*.*.*`.

```
on:
  push:
    tags: ['v*.*.*']
```

Сборка:
1. Checkout репозитория
2. Настройка JDK 17 (temurin)
3. Настройка Android SDK
4. `npm ci` + `npx cap sync android`
5. `./gradlew assembleRelease` (или assembleDebug если нет keystore)
6. Загрузка APK как ассет релиза через `gh release create`

### Шаг 2: Создать `.gitignore` дополнения

Убедиться что `android/.gradle`, `android/build`, `android/app/build` в `.gitignore` (build-артефакты не в репозитории).

### Шаг 3: Использование

```bash
# Собрать и выпустить:
git tag v1.0.0
git push origin v1.0.0
# Actions соберёт APK и опубликует релиз автоматически
```

## Part 2: In-App Update Checker

### Шаг 4: Добавить скрипт `www/update-checker.js`

Скрипт проверяет GitHub API при загрузке меню:

1. GET `https://api.github.com/repos/tewiqpy-ai/tictactoe-game/releases/latest`
2. Извлечь `tag_name` (например "v1.1.0") и `assets[].browser_download_url" для APK
3. Сравнить с текущей версией из `capacitor.config.json` или захардкоженной константы
4. Если версия новее — показать красивый модал "Доступно обновление!" с:
   - Номером новой версии
   - Кнопкой "Обновить"
   - Кнопкой "Позже"
5. При нажатии "Обновить" — открыть URL скачивания APK (браузер скачает файл, Android предложит установить)

### Шаг 5: Подключить скрипт в `www/menu.html`

Добавить `<script src="update-checker.js"></script>` перед закрывающим `</body>`.

### Шаг 6: Синхронизация

```bash
npx cap sync android
```

---

## Part 3: Версионирование

### Шаг 7: Согласовать версии

- `package.json` version: `1.0.0`
- `android/app/build.gradle` versionCode: `1`, versionName: `1.0.0`
- Скрипт обновления читает version из `capacitor.config.json` или специального файла

## Файлы

| Файл | Действие |
|---|---|
| `.github/workflows/release.yml` | Создать |
| `www/update-checker.js` | Создать |
| `www/menu.html` | Изменить (добавить script) |
| `android/app/build.gradle` | Изменить (версия) |
| `package.json` | Изменить (версия) |
| `.gitignore` | Проверить/дополнить |

## Порядок выполнения

1. Переавторизация `gh auth login`
2. Создать GitHub Actions workflow
3. Создать update-checker.js
4. Подключить в menu.html
5. Согласовать версии
6. Закоммитить и запушить
7. Затегать и создать первый релиз

## Verification

- Push тега → Actions запускается → APK публикуется как asset релиза
- Открыть приложение → при наличии нового релиза показывается диалог обновления
- Кнопка "Обновить" открывает ссылку на скачивание APK
